## 实施计划：Qwen3.5 DiffusionVL（推理先行）

### 背景：Qwen3.5-4B 架构（与 Qwen3-VL 的关键差异）

| 维度 | Qwen3-VL | Qwen3.5-4B |
|---|---|---|
| 注意力 | 纯 Full (28层) | **混合**：24 线性(Gated DeltaNet) + 8 全注意力(每4层1个) = 32层 |
| Dense/MoE | Dense | Dense |
| 视觉端 | ViT + **DeepStack** | ViT，**无 DeepStack**（deepstack_visual_indexes=[]） |
| MTP | 无 | 有（推理忽略） |
| Cache | DynamicCache (纯 KV) | DynamicCache (混合：线性层 conv_states+recurrent_states，全注意力 KV) |
| vocab/embed | 151936 | **248320** |
| hidden_size | 2048 | **2560** |

### 核心技术挑战与方案

**挑战**：BD3-LM 逐块去噪需要 `store_kv=False`（反复评估同一 block 不提交缓存），但 Qwen3.5 的线性注意力层**没有只读模式** —— cache_params 非 None 时强制写入。

**方案：快照-恢复**（我们之前推导的）
- block 去噪前：快照所有线性层的 `conv_states` + `recurrent_states`（24层 × 2 个 tensor）
- 每次 step forward（会写入 cache）
- 下一次 step 前：`.copy_()` 恢复快照
- block 完成（store_kv=True）：用最终揭晓的 embeds 正式更新 cache

全注意力层（8层）仍用现有 store_kv=False 逻辑（读缓存拼接不写入）。

##### `modeling_diffusionvl_qwen3_5.py`（核心，~1200 行）

从 transformers 导入 Qwen3.5 骨干类：
```python
from transformers.models.qwen3_5.modeling_qwen3_5 import (
    Qwen3_5TextModel, Qwen3_5Attention, Qwen3_5GatedDeltaNet,
    Qwen3_5DecoderLayer, Qwen3_5PreTrainedModel,
    Qwen3_5VisionModel, Qwen3_5VisionPatchMerger,
)
```

关键类：
- `DiffusionVLQwen3_5Attention(Qwen3_5Attention)`：is_causal=False + store_kv hook（全注意力层）
- `DiffusionVLQwen3_5TextModel(Qwen3_5TextModel)`：
  - 替换全注意力层的 self_attn 为 DiffusionVL 版本（is_causal=False）
  - **线性注意力层保持不变**（Gated DeltaNet 不需要 is_causal 修改 —— 它的递归本身就是因果的）
  - 接受 dict attention_mask（full_attention/linear_attention 两种 key）
  - 加 cache 快照/恢复方法：`snapshot_linear_cache()` / `restore_linear_cache()`
- `DiffusionVLQwen3_5VisionTower`：包装 Qwen3_5VisionModel，在 merger 前停止（与 Qwen3-VL 类似但无 DeepStack）
- `DiffusionVLQwen3_5Projector`：主 merger
- `DiffusionVLQwen3_5Model`：内层模型（vision_tower + mm_projector + text model）
- `DiffusionVLQwen3_5ForConditionalGeneration`：
  - `generate` → prepare_inputs_labels_for_multimodal → generate_with_bd3lm
  - **generate_with_bd3lm 的核心改动**：
    - 全注意力层：store_kv=False 时读缓存拼接（现有逻辑）
    - 线性层：**快照-恢复**模式（block 去噪前 snapshot，每次 step 后 restore）
  - `prepare_inputs_labels_for_multimodal`：与 Qwen3-VL 版类似但无 DeepStack（返回 6 元组）

**generate_with_bd3lm 伪代码**：
```python
for block_idx in range(生成区):
    # 快照线性层 cache
    linear_snapshot = snapshot_linear_cache(past_key_values)
    
    for step in range(steps):
        # 恢复线性层 cache 到 block 开始状态
        restore_linear_cache(past_key_values, linear_snapshot)
        
        # forward（全注意力层 store_kv=False，线性层会写入但下次 step 前恢复）
        outputs = self.model(cur_block_embeds, past_key_values, store_kv=False)
        logits = lm_head(outputs)
        # 揭晓 token...
    
    # block 完成，用最终 embeds 正式更新（store_kv=True）
    restore_linear_cache(past_key_values, linear_snapshot)  # 先恢复
    self.model(cur_block_embeds, past_key_values, store_kv=True)  # 正式写入
```

### 与 Qwen3-VL 版的关键简化

| 方面 | Qwen3-VL 版 | Qwen3.5 版 |
|---|---|---|
| DeepStack | 有（vision + LM 注入 + cursor 逻辑） | **无**（简化大量代码） |
| prepare_inputs 返回 | 8 元组（含 visual_pos_masks + deepstack） | **6 元组**（标准 LLaVA 风格） |
| cache 快照 | 不需要（全注意力 store_kv=False 足够） | **需要**（线性层 cache 快照-恢复） |
